# Patterna Output Guide

This guide explains the main output folders generated by Patterna and what each file type is used for.

Patterna generates both image-based quality-control outputs and quantitative tables. Some outputs are meant for visual inspection, while others are intended for downstream analysis, figure generation, or biological interpretation.

## Output folder overview

A typical Patterna output directory may contain:

```text
masks/
grids/
profiles/
heatmaps/
topography_maps/
debug/
qc/
cells/
metrics/
excel/
grouped_profiles/
grouped_heatmaps/
grouped_topography_maps/
grouped_orthogonal/
cell_phenotypes/
cell_intensity_maps/
cell_phenotype_maps/
grouped_cell_intensity_maps/
summary_panels/
```

Not every folder will always contain files. Some outputs depend on whether specific settings are enabled, such as nuclei segmentation, phenotype classification, grouped plots, or integrated summary panels.

---

## `masks/`

Contains binary colony masks.

These masks define the region of the micropatterned colony used for spatial sampling. The mask is used to determine the colony centroid and the true boundary along each angular direction.

Typical files:

```text
sample_name_mask.tif
```

Use this folder to check whether Patterna correctly detected the colony area.

---

## `grids/`

Contains the main spiderweb-sampled intensity arrays.

Each grid stores the fluorescence intensity data after converting the colony into normalized spatial coordinates.

Typical files:

```text
sample_name_grid.npy
```

Grid shape:

```text
channel × radius × angle
```

Meaning:

```text
channel = fluorescence channel
radius  = normalized distance from colony center to edge
angle   = angular sampling direction
```

These files are useful for advanced downstream analysis or regenerating plots without rerunning the full image-processing pipeline.

---

## `profiles/`

Contains radial and orthogonal intensity profiles.

Typical files may include:

```text
sample_name_radial_profile.svg
sample_name_radial_profile_channelmaxnorm.svg
sample_name_radial_profiles.csv
sample_name_orthogonal_profiles.svg
sample_name_orthogonal_profiles_curves.csv
```

Main outputs:

* Radial profiles show mean marker intensity from colony center to edge.
* Channel-max-normalized profiles show each channel scaled to its own maximum.
* Orthogonal profiles compare marker organization along the major and minor colony axes.
* CSV files contain the numerical profile values used to generate the plots.

Use this folder when comparing marker distribution patterns across radius or across axes.

---

## `heatmaps/`

Contains rectangular angle-radius heatmaps.

Typical files:

```text
sample_name_spiderweb_ch0.svg
sample_name_spiderweb_ch1.svg
sample_name_topography_ch1.svg
sample_name_topography_composite_rect.png
```

These maps show spatial organization in the normalized spiderweb coordinate system.

Axes:

```text
x-axis = angle
y-axis = normalized radius from center to edge
```

Use these outputs to inspect whether marker expression is spatially uniform, radial, asymmetric, or localized to specific angular regions.

---

## `topography_maps/`

Contains polar topography visualizations.

Typical files:

```text
sample_name_polar_topography_ch1.pdf
sample_name_polar_topography_ch1.png
sample_name_topography_composite_polar.png
```

These files project the spiderweb-sampled data into a circular polar-style representation.

Use these maps for visualizing spatial marker patterns in a format that resembles the original colony geometry while still using normalized coordinates.

---

## `debug/`

Contains debugging and quality-control images related to channel handling and spiderweb sampling.

Typical files:

```text
sample_name_RAW_CHANNEL_QC.png
sample_name_CLEAN_after_BF_removal_RAW_CHANNEL_QC.png
sample_name_spiderweb_overlay.png
```

Use this folder to check:

* Whether channels were loaded correctly.
* Whether a brightfield channel was removed correctly.
* Whether the spiderweb sampling overlay matches the colony geometry.
* Whether the mask and radial sampling are visually reasonable.

This folder is especially useful when troubleshooting unusual images or unexpected outputs.

---

## `qc/`

Contains nuclei segmentation quality-control overlays.

Typical files:

```text
sample_name_nuclei_boundaries.png
sample_name_nuclei_colored.png
```

These files are generated when single-cell analysis is enabled.

Use this folder to inspect whether nuclei were segmented correctly before interpreting per-cell measurements or phenotype classification.

Important: Always check these QC images before trusting cell-level results.

---

## `cells/`

Contains per-cell measurement tables and instance-label masks.

Typical files:

```text
sample_name_cells.csv
sample_name_labels.tif
```

The CSV files contain one row per segmented nucleus or cell object.

Common columns include:

```text
cell_id
x
y
r_frac
theta_rad
theta_deg
ring_bin
sector_bin
area_px
eccentricity
solidity
ch0_mean
ch1_mean
ch2_mean
ch3_mean
ch0_p90
ch1_p90
ch2_p90
ch3_p90
```

These outputs are used for single-cell marker quantification, radial cell-position analysis, and phenotype classification.

---

## `metrics/`

Contains combined quantitative summary tables.

Typical files:

```text
metrics.csv
topography_summary.csv
orthogonal_profiles.csv
pattern_fingerprints.csv
fingerprint_summary.csv
cells_all.csv
```

Main files:

### `metrics.csv`

Basic per-image, per-channel spatial metrics.

May include:

```text
edge_mean
center_mean
edge_center_ratio
radius_corr
edge_angular_asymmetry
```

### `topography_summary.csv`

Summarizes radial signal distribution per channel.

May include:

```text
frac_inner
frac_mid
frac_outer
r_mean
r_peak
r_width
```

### `orthogonal_profiles.csv`

Summarizes major/minor axis comparisons.

May include:

```text
edge_major
edge_minor
center_major
center_minor
AI_major_minor
major_peak_r
minor_peak_r
```

### `pattern_fingerprints.csv`

Detailed spatial fingerprint table for each image and channel.

### `fingerprint_summary.csv`

Simplified spatial fingerprint summary, including dominant radial region and key spatial descriptors.

### `cells_all.csv`

Combined per-cell table across all processed images.

Use this folder for downstream statistics, figure generation, and biological interpretation.

---

## `excel/`

Contains organized Excel workbooks for radial profiles.

Typical files:

```text
Patterna_radial_profiles_by_channel_makis_normalized.xlsx
Patterna_radial_profiles_by_channel_global.xlsx
```

Each workbook contains separate sheets by channel.

These workbooks are useful for comparing replicate micropatterns side by side and for generating grouped plots manually if needed.

Two common profile types are exported:

### Makis-normalized profiles

Each channel curve is normalized to its own maximum.

Useful for comparing curve shape independent of absolute intensity.

### Global profiles

Uses the Patterna grid-normalized profile.

Useful for preserving more of the relative differences between channels or samples.

---

## `grouped_profiles/`

Contains grouped radial profile plots across multiple micropatterns.

Typical files:

```text
grouped_radial_profiles_all_channels_curve_max_normalized.svg
grouped_radial_profiles_all_channels_grid_0to1_normalized.svg
grouped_radial_profile_ch1_TBXT_curve_max_normalized.svg
```

These plots show replicate-level mean profiles with variability, usually SD or SEM depending on the selected setting.

Use these outputs for comparing marker trends across multiple micropatterns.

---

## `grouped_heatmaps/`

Contains grouped rectangular heatmaps averaged across micropatterns.

Typical files:

```text
grouped_rectangular_heatmap_ch1_TBXT_n3.svg
```

These maps summarize the average angle-radius organization for each channel across all compatible grids in the run.

---

## `grouped_topography_maps/`

Contains grouped polar heatmaps averaged across micropatterns.

Typical files:

```text
grouped_polar_heatmap_ch1_TBXT_n3.pdf
grouped_polar_heatmap_ch1_TBXT_n3.png
```

These outputs are useful for visualizing average spatial organization across multiple colonies.

---

## `grouped_orthogonal/`

Contains grouped major/minor axis profile plots.

Typical files:

```text
grouped_orthogonal_ch1_TBXT_n3.svg
```

These plots compare averaged marker profiles along the major and minor axes across multiple micropatterns.

Use these outputs to assess anisotropy or directionality in marker expression.

---

## `cell_phenotypes/`

Contains phenotype classification outputs.

Typical files:

```text
Patterna_cell_phenotype_classification.xlsx
cell_phenotype_stacked_by_ring.svg
cell_phenotype_stacked_by_image.svg
positive_fraction_by_marker_ring.svg
marker_intensity_threshold_diagnostics.svg
marker_intensity_threshold_diagnostics.csv
```

The Excel workbook may include:

```text
classified_cells
summary_by_image
summary_by_ring
summary_by_channel
thresholds
```

Use this folder to inspect cell phenotype classes, marker-positive fractions, radial phenotype organization, and threshold diagnostics.

Important: Phenotype classification depends strongly on the selected thresholds. Percentile-based thresholds are exploratory unless validated with proper controls.

---

## `cell_intensity_maps/`

Contains cell-level marker intensity maps.

Typical files:

```text
sample_name_cell_intensity_TBXT.svg
sample_name_cell_intensity_SOX2.svg
sample_name_cell_intensity_GATA6.svg
cell_intensity_display_scales.csv
```

Each map colors segmented cells by marker intensity.

Use this folder to visually inspect single-cell marker expression patterns across the colony.

The display scale CSV records the intensity display limits used for each marker.

---

## `cell_phenotype_maps/`

Contains standalone phenotype maps.

Typical files:

```text
sample_name_cell_phenotype_map.svg
sample_name_cell_phenotype_map.png
```

Each segmented cell is colored according to its classified phenotype, such as:

```text
negative
TBXT only
SOX2 only
GATA6 only
TBXT+SOX2
TBXT+GATA6
SOX2+GATA6
TBXT+SOX2+GATA6
```

Use this folder to visually inspect spatial organization of cell phenotypes.

---

## `grouped_cell_intensity_maps/`

Contains grouped disk-rendered marker maps.

Typical files:

```text
grouped_disk_marker_map_TBXT_n3.svg
grouped_disk_marker_map_SOX2_n3.svg
grouped_disk_marker_map_GATA6_n3.svg
```

These maps are based on averaged spiderweb grids and show mean marker organization across multiple micropatterns in a disk-like format.

---

## `summary_panels/`

Contains integrated spatial summary panels.

Typical files:

```text
sample_name_integrated_spatial_summary.svg
sample_name_integrated_spatial_summary.png
```

These panels combine multiple outputs into one figure, such as:

* Cell intensity maps
* Cell phenotype map
* Marker-positive fraction by radial zone
* Phenotype distribution by radial zone
* Phenotype class distribution
* Radial intensity profiles
* Marker-positive fraction across the whole micropattern
* Major/minor axis visualization
* Anisotropy information
* Basic metadata summary

Use this folder for high-level inspection and presentation-ready summaries.

---

## Recommended QC workflow

Before interpreting the final metrics, inspect outputs in this order:

1. `debug/`

   * Check raw and cleaned channels.
   * Confirm brightfield removal if applicable.
   * Check spiderweb overlay.

2. `masks/`

   * Confirm the colony mask matches the micropattern.

3. `qc/`

   * Confirm nuclei segmentation quality.

4. `profiles/` and `heatmaps/`

   * Check whether spatial marker patterns look biologically plausible.

5. `metrics/`

   * Use quantitative summaries only after visual QC passes.

6. `cell_phenotypes/`

   * Inspect threshold diagnostics before interpreting positive/negative phenotype classes.

---

## Notes

Patterna outputs should be interpreted together with image-level QC. Quantitative metrics are only meaningful if the colony mask, channel mapping, and nuclei segmentation are correct.

Phenotype classification is exploratory unless thresholds are calibrated with appropriate controls.
