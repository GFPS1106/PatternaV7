# Test Data and Example Outputs

The microscopy test data and full Patterna example outputs are hosted externally because OME-TIFF files and generated image-analysis output folders can exceed GitHub upload size limits.

## External data folder

Download the test data and example outputs here:

[Patterna test data and example outputs — Google Drive](https://drive.google.com/drive/folders/1BQD78u8Z2A6_fnVph11QxIqpqWnS4Myq?usp=sharing)

The Google Drive folder contains:

```text
PatternaV7_test_data_and_example_outputs/
├── test_data/
│   ├── CTRL_6.tif
│   └── CTRL_10.tif
└── example_outputs/
    └── PatternaV7_test_for_repository/
```

## Contents

### `test_data/`

This folder contains two example OME-TIFF microscopy images of micropatterned stem-cell colonies.

These images can be used to test whether Patterna runs correctly and generates the expected output structure.

### `example_outputs/`

This folder contains the full output generated by running PatternaV7 on the provided test images.

These outputs can be used as a reference for what a successful Patterna run should produce.

## How to use the external test data

1. Download the Google Drive folder.
2. Copy or move the contents of `test_data/` into the local `test_data/` folder in this repository.
3. Open `src/PatternaV7.py`.
4. Update the input and output paths:

```python
INPUT_DIR = r"path/to/test_data"
OUT_DIR = r"path/to/Patterna_output"
```

5. Run Patterna:

```bash
python src/PatternaV7.py
```

6. Compare the newly generated output folder with the provided `example_outputs/` reference folder.

## Expected channel mapping

The current default configuration assumes the following cleaned channel mapping:

```text
clean ch0 = DAPI
clean ch1 = TBXT
clean ch2 = SOX2
clean ch3 = GATA6
```

If a 5-channel image is detected, Patterna removes the brightfield channel and keeps the cleaned fluorescence stack for analysis.

## Notes

* The external files are provided for testing and demonstration purposes.
* Large raw microscopy files and full generated outputs are not stored directly in GitHub to keep the repository lightweight.
* If the Google Drive link becomes unavailable, please contact the repository author.
* Always inspect the QC outputs before interpreting metrics or phenotype classifications.
